# ResumeForge - Professional Resume Builder System

## Project Overview
ResumeForge is a comprehensive web application designed to help users create, manage, and export professional resumes. Built using Java Jakarta EE (Servlets/JSP) for the backend and React with Tailwind CSS for a premium, high-fidelity frontend experience.

## Technical Stack
- **Backend**: Java Jakarta EE (Servlets), PostgreSQL
- **Frontend**: React 18, Tailwind CSS, Lucide-style SVG icons
- **Architecture**: MVC (Model-View-Controller)
- **Styling**: Glassmorphism, Dark Mode, Premium HSL Color Palettes

---

## Core Features & Functionality

### 1. User Authentication
- **Registration**: Allows new users to create accounts.
- **Login**: Secure access for existing users with session management.
- **Logout**: Safe termination of user sessions.

### 2. Resume CRUD Operations
The system supports full CRUD (Create, Read, Update, Delete) management for resumes:
- **Create**: A 6-step guided form to build a resume from scratch.
- **Read**: 
    - **Dashboard**: View a list of all saved resumes.
    - **Preview**: High-fidelity rendering of the resume in different templates.
- **Update**: Edit any existing resume to update details, education, experience, etc.
- **Delete**: Remove obsolete resumes from the database.
- **Duplicate**: Clone an existing resume to create a new version quickly.

### 3. Advanced Search & Filtering
The dashboard includes a powerful search system implemented in `ResumeDAO.searchResumes`:
- **Text Search**: Search by title, full name, or email (characters and numbers).
- **Date Filtering**: Filter resumes based on the exact date they were created/updated.
- **Time Range**: Select a "Start Time" and "End Time" to filter by the precise time of creation.
- **Persistent State**: Search results and filter inputs stay active after navigation for a better user experience.

### 4. Interactive Resume Form
The resume building process is divided into 6 logical steps:
1. **Personal Info**: Name, contact, social links, and summary.
2. **Education**: Institutions, degrees, and dates.
3. **Experience**: Work history and job descriptions.
4. **Skills**: Technical and soft skills with proficiency levels.
5. **Projects**: Showcase key work with descriptions and URLs.
6. **Review & Submit**: A final summary step to verify all data before committing to the database.

## Technical Details

### Database Schema (PostgreSQL)
- `users`: Stores user credentials and profile data.
- `resumes`: Primary data for the resume (title, template, contact info).
- `education`: Linked to `resumes` via `resume_id`.
- `experience`: Linked to `resumes` via `resume_id`.
- `skills`: Linked to `resumes` via `resume_id`.
- `projects`: Linked to `resumes` via `resume_id`.

### Key Controllers
- `DashboardController`: Handles listing, searching, and filtering of resumes.
- `ResumeFormController`: Manages the multi-step form state and database persistence.
- `ResumeController`: Utility for actions like delete and duplicate.

## How to Run & Deploy
1. Ensure **PostgreSQL** is running and the `resume_db` is created.
2. Configure `DBConnection.java` with your database credentials.
3. Deploy the project folder to the Apache Tomcat `webapps` directory.
4. Access via `http://localhost:8080/Eresume`.
